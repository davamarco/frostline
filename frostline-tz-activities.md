## 1. Что это за блок

Секция на главной под названием **WHAT YOU CAN DO**. Три карточки в ряд, расположенные «лесенкой» (на разной высоте), слегка наезжающие друг на друга. Из верхнего края каждой карточки «вылезает» вырезанный сноубордист (PNG с прозрачным фоном). Средняя карточка — тёмная, для контраста. Вся карточка кликабельна и ведёт на другую страницу.

Стиль: палитра белый / салатовый `--lime` / лимонный `--lemon` / тёмный `--ink`, заголовки шрифтом Anton капсом. (Переменные уже заданы в `style.css`.)

## 2. Раскладка

- Заголовок секции по центру: **WHAT YOU CAN DO**, под ним тонкая лимонная линия-акцент.
- Под заголовком — ряд из 3 карточек (`display: flex`, gap ~16–20px, `align-items: flex-start`).
- **Стаггер по высоте** через разный `margin-top`: левая 0, средняя ~54px (ниже всех), правая ~28px.
- Карточки: `border-radius: 22px`, `padding-top` достаточный (~90px), чтобы под «вылезающим» райдером было место. `overflow: visible` (чтобы райдер мог выходить за край).
- У секции сверху отступ ~90px, чтобы верхние райдеры не обрезались.
- **Адаптив:** на узких экранах (< ~560px) карточки в столбик, `margin-top` сбросить, gap увеличить (~60px), чтобы райдеры не накладывались.

## 4. Кликабельность

- **Вся карточка** — это ссылка `<a>` (оборачиваем всю карточку), курсор `pointer`.
- Заголовок «факт» и описание — внутри ссылки, без отдельных href.

## 5. Анимация появления при прокрутке (главное!)

Цель — «премиальное» появление: карточки выезжают **по очереди**, снизу вверх, с плавным замедлением.

Реализация на ванильном JS через `IntersectionObserver`:

1. Карточки в исходном состоянии: `opacity: 0; transform: translateY(40px);`.
2. Когда секция попадает в зону видимости — добавляем класс `.is-visible`, который возвращает `opacity: 1; transform: translateY(0)`.
3. **Stagger:** задержки между карточками заданы через `transition-delay`:
   - карточка 1 — `0s`
   - карточка 2 — `0.12s`
   - карточка 3 — `0.24s`
4. **Easing (важно для «дорогого» ощущения):** `transition: opacity .6s, transform .7s cubic-bezier(0.22, 1, 0.36, 1);`
5. **Райдеры появляются чуть позже карточек** — отдельный элемент райдера со своей анимацией и `transition-delay` примерно на `0.15–0.2s` больше, чем у его карточки. Эффект: сначала карточка, потом на неё «запрыгивает» сноубордист (лёгкий выезд сверху + прозрачность).
6. Срабатывает **один раз** (после первого показа `observer.unobserve` или флаг), чтобы при прокрутке туда-сюда не дёргалось.
7. **Обязательно:** при `@media (prefers-reduced-motion: reduce)` — все эти анимации отключить, карточки и райдеры сразу видимы (`opacity:1; transform:none`).

## 7. Картинки для этого блока (генерировать)

Нужны **3 PNG с ПРОЗРАЧНЫМ фоном** — вырезанные фигуры сноубордистов (без квадратного фона вокруг):

| Файл | Что на нём |
|---|---|
| `rider-freeride.png` | сноубордист в динамичной позе, едет/режет склон, вид сбоку |
| `rider-park.png` | сноубордист в прыжке, схватил доску (grab), эффектная поза |
| `rider-lessons.png` | сноубордист стоит уверенно, дружелюбная поза, можно с поднятой рукой |

Как получить прозрачный фон:
- в ChatGPT добавь в промт: `full body snowboarder, dynamic pose, isolated on transparent background, PNG cutout, no background, studio lighting`
- **или** сгенерь обычное фото и убери фон на **remove.bg** (бесплатно).

Ориентация — вертикальная (portrait). После — сжать (TinyPNG поддерживает PNG с прозрачностью).

> Пока PNG нет — пусть Claude Code поставит вместо райдеров временные силуэты или цветные заглушки нужной формы, чтобы видеть раскладку и анимацию.
